# Specs-First Curriculum Design

CARE, EATP, CO, COC, and PACT are first principles that generate the curriculum. They are not post-hoc lenses applied to existing material.

## MUST

- Every module learning outcome maps to one or more specification concepts
- Framework choices are derived from specs, not the other way around
- Case analyses start from spec principles, not from the case narrative
- Assessments test specification application, not specification recall

## MUST NOT

- Teach specs as "theory" separate from practice — specs ARE the practice
- Present DX frameworks as standalone models — they derive from CARE/EATP/CO
- Introduce Kailash tools without connecting to the spec concept they implement
- Use specs as post-hoc labels ("this is an example of Dual Plane") — instead derive from specs ("Dual Plane tells us to classify this as...")

**Why:** Post-hoc labeling produces students who can name concepts but cannot generate solutions from principles. Specs-first produces practitioners who derive correct approaches from first principles in novel situations.
